# 《微光深淵》(Gloamdepths)玩法設計文件

一款受 **Core Keeper** 啟發、但玩法全新的 2D 俯視角地底合作生存遊戲。
網頁版(HTML5 Canvas + 原生 JavaScript + PeerJS 連線),雙擊 `index.html` 即可遊玩,支援 1–4 人連線。

---

## 一、與 Core Keeper 的差異(原創核心機制)

Core Keeper 是「自由沙盒 + 打 Boss」;本作把它改造成 **「光即生命」的守護戰**:

| | Core Keeper | 微光深淵(本作) |
|---|---|---|
| 核心目標 | 自由探索、打 Boss | **守護會耗能的「星核」**,找回 3 塊星核碎片使其甦醒 |
| 黑暗 | 只是看不見 | 黑暗會湧出「蝕影」;**光照範圍 = 你的領土** |
| 生存壓力 | 飢餓值 | **星核能量持續下降**,要挖「光晶」回來餵,歸零即全隊失敗 |
| 防禦 | 幾乎不需要 | **定期「暗潮」圍攻星核**,要建牆、蓋光塔守家(輕度塔防) |
| 多人 | 一起玩 | 玩法天生鼓勵**分工**:有人挖礦補給、有人守家建設、有人遠征找碎片 |

## 二、遊戲流程

```
開局:星核微亮,能量緩慢流失
 ↓
近程循環:挖光晶餵星核、採銅木、建工作台/熔爐、做銅裝
 ↓
第一波暗潮(開局 4 分鐘後,之後每 4 分鐘一波、逐波變強)
 ↓
中程:鐵裝、用木牆/石牆+光塔把基地築成堡壘
 ↓
遠征:三座「守衛神殿」分佈在地圖外圈(小地圖有標記),
      擊敗石像守衛 ×3,帶回星核碎片
 ↓
終局:放入第 3 塊碎片 → 觸發「最終暗潮」大圍攻 → 撐過即通關
```

## 三、世界設計

- 程序化生成 280×280 格地圖(細胞自動機洞穴 + 保底隧道),每次新開隨機種子(2026-07-15 由 200×200 放大,解決「探索太快就沒東西可挖」)。
- 以星核為中心的環形難度分區(半徑集中定義在 `ZONE_R`,config.js):

| 區域 | 距中心 | 牆壁 | 資源 | 敵人 |
|------|--------|------|------|------|
| 星核區 | 0–8 | 安全區、自帶光 | — | 不生怪 |
| 泥土區 | 8–59 | 泥土牆 | 銅礦、光晶、木根、螢光蘑菇 | 小蝕影、爆裂蝕影 |
| 石岩區 | 59–101 | 石牆 | 鐵礦、光晶 | 蝕影獵手、吐影者、穿牆幽影 |
| 黑曜區 | 101–134 | 黑曜岩(需鐵鎬) | 金礦、光晶、**三座神殿** | 深淵蝕影、裂地者、石像守衛 |
| 淵核區(通關解封) | 134–162 | 淵岩(需金鎬) | 鑽石、光晶、淵晶 | 淵魂、蝕裂者 |
| 基岩 | 162+ | 不可破壞 | — | — |

- **怪物只在黑暗處生成**:火把與光塔照亮的區域就是安全區,把光推出去 = 擴張領土。各分區怪物是「加權池」而非單一固定種類(`ZONE_SPAWN_POOL`,config.js),招牌怪占多數、混入同分區調性的變化款。

## 四、星核(團隊命脈)

- 能量上限 100,每秒流失 0.25(約 6.5 分鐘耗盡)。
- 站在星核旁按 **F**,把身上的光晶全部灌入(每顆 +6)。
- 暗潮怪物攻擊星核會直接扣能量。
- **能量歸零 = 全隊失敗**(30/15/5 會警告)。
- 集滿 3 塊碎片並撐過最終暗潮 → 星核甦醒,通關。

## 五、進度系統(工具階級)

| 階級 | 鎬 | 劍 | 護甲 | 取得 |
|------|----|----|------|------|
| 木 | 木鎬(初始) | 木劍(初始) | — | 開局自帶 |
| 銅 | 銅鎬 | 銅劍 | — | 泥土區銅礦 → 熔爐 |
| 鐵 | 鐵鎬 | 鐵劍 | 鐵甲 −30% | 石岩區鐵礦(需銅鎬) |
| 金 | 金鎬 | 金劍 | 金甲 −50% | 黑曜區金礦(需鐵鎬) |

挖掘與攻擊**自動使用背包中最好的鎬/劍**,快捷欄留給放置物與食物,操作單純。

## 六、建造與防禦(輕度塔防)

| 建築 | 功能 |
|------|------|
| 火把 | 照明(半徑 7 格),阻止黑暗生怪 |
| 木牆 / 石牆 | 擋路;暗潮怪會啃牆,石牆耐久高得多 |
| 光塔 | 照明 + **自動攻擊** 5.5 格內的蝕影 |
| 工作台 / 熔爐 | 合成站(靠近才能合成對應配方) |

暗潮怪從黑暗處出現、直線衝向星核、會啃穿擋路的牆 —— 用地形、牆與光塔設計防線。

## 七、生存要素(輕量)

- HP 100;10 秒未受傷後緩慢回血。
- 食物:螢光蘑菇 +15、烤蘑菇 +40(蘑菇會在泥土區緩慢重生)。
- 死亡:5 秒後在星核重生、物品不掉落(休閒向,鼓勵朋友同樂)。

## 八、多人連線設計(核心需求)

### 架構:PeerJS(WebRTC P2P)+ 房主權威

```
房主瀏覽器 = 伺服器(跑全部遊戲邏輯:怪物、暗潮、合成判定、星核)
     ↑ 玩家輸入/動作(位置、挖礦、攻擊、合成…)
     ↓ 世界快照 10 次/秒 + 地形變更事件
朋友瀏覽器 = 客戶端(自己的移動在本地預測,其餘顯示房主廣播的狀態)
```

- **開房**:房主按「開房」,拿到 5 碼房號傳給朋友。
- **加入**:朋友輸入房號即連線,**不需要架設或租用任何伺服器**(PeerJS 提供免費信令服務,遊戲資料走瀏覽器對瀏覽器直連)。
- **存檔在房主**(你指定的需求):世界、星核、每位玩家的背包都存在**房主的瀏覽器 localStorage**;朋友的背包以「玩家名字」為鍵一併保存,下次用同名加入同一位房主的存檔,就能拿回自己的裝備。
- 自動存檔:每 30 秒 + 房主關閉頁面時。
- 沒網路時仍可離線單人遊玩(同一套程式,只是不啟用連線)。

### 已知限制(P2P 的取捨)
- 房主關頁面 = 這局結束(存檔還在,下次再開房繼續)。
- 極少數嚴格 NAT 網路環境可能連不上(需 TURN 中繼,屬未來擴充)。
