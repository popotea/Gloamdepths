# 《微光深淵》未來開發規劃

這份文件整理目前已有的系統範圍、以及規劃中的新功能——每項都附上「為什麼適合」「大概工程量」「跟其他系統的相依關係」「開工前要先想清楚的設計問題」。用來決定優先順序,不是寫死的排程,隨時可以調整。

---

## 目前已有系統(基準線,避免規劃重複)

- **核心循環**:星核能量流失/餵光晶、暗潮圍攻、三神殿石像守衛、3 塊碎片、最終暗潮通關
- **採集**:木/銅/鐵/金/鑽石鎬分級、砂礫/煤/鑽石礦、光晶、木根
- **戰鬥**:劍(自動)、矛(手動/長距離)、鎚(手動/範圍+擊退)、弓/弩(彈藥)、光晶法杖(貫穿);焰/冰/光/重擊 4 種元素相剋
- **敵人**:9 種蝕影(含成群/遠程/自爆/穿牆/拆牆特性)+ 1 隻真 Boss(石像守衛)
- **防禦建築**:火把、木/石牆、光塔(自動)、箭塔(彈藥制+每人數量上限);**已放置的建築(工作台/熔爐/光塔/箭塔/火把)左鍵敲幾下可拆回背包道具,能帶去別處重新放置**
- **據點**:神殿(Boss)、廢墟(寶箱)、蝕影巢穴(持續生怪,可拆除,2026-07 起分三種:一般/孢群/精英)
- **成長**:等級/經驗(單純 +HP/+攻擊%)、強化卷軸(+1~+5 衝裝)、體力/衝刺
- **農耕**:鏟子翻土、種子、三階段成長、自動收成(2026-07 剛完成)
- **多人**:PeerJS 房主權威連線、存檔在房主、玩家背包依名字保存、聊天(含歷史紀錄可 ↑/↓ 叫回)
- **秘笈選單** `/power`:全地圖開亮、召喚怪物、星核/暗潮控制等除錯指令

## 一、短期(小工程,直接接在現有系統上)

### 1.1 料理擴充
- **現況**:熟食只有「烤蘑菇」一種
- **做法**:光傘菇料理、礦物入菜,做出短時 buff(移速/防禦/挖礦速度),而不只是回血
- **工程量**:小,`RECIPES` + `ITEMS` 加幾筆,加一個 `buff` 效果欄位跟對應的 timer 處理(玩家身上加一個 `p.buffs` 陣列)
- **相依**:無,獨立擴充

### 1.2 圍籬/柵欄
- **現況**:暗潮怪會啃穿任何非戰鬥區的建築,農地完全無防護
- **做法**:類似木牆但矮一階的圍籬,擋怪但不擋玩家視線/子彈
- **工程量**:小,新增一個 tile 或 object 類型,行為照抄 `T.WOODWALL`
- **相依**:讓農耕系統更完整,建議跟農耕綁在一起考慮,但可獨立上線

### 1.3 統計面板
- **做法**:擊殺數、存活時間、暗潮波數、最高等級,存進存檔,ESC 選單可查看
- **工程量**:小,`G` 加幾個計數器,`updateEnemies`/`updateWave` 順手累加,`buildSave`/`applySave` 一起存
- **相依**:無

### 1.4 大地圖(M 鍵開全螢幕地圖 + 滾輪縮放)
- **現況**:右上角小地圖是固定 200×200 像素的縮圖(1 像素=1 格),遠處的建築標記(神殿/巢穴)只有一個小色塊,看不清楚地形細節
- **做法**:按 M 開一個蓋滿全螢幕的地圖面板,直接把現有 `drawMinimap()` 的邏輯放大畫在大 canvas 上;滑鼠滾輪縮放(調整每格畫幾個像素)、可能要加拖曳平移(縮到夠大時,200×200 格會超出螢幕寬度);建築標記在大地圖上可以顯示更完整的資訊(圖示+名稱,而不是只有色塊),滑鼠移到標記上可以顯示 tooltip(如「精英巢穴」「石像守衛已擊敗」)
- **工程量**:小中,主要是渲染邏輯要能重複使用(現有小地圖跟新的大地圖共用同一份資料來源與畫法,只是縮放比例、canvas 大小、互動邏輯不同),另外滾輪縮放+拖曳平移的座標轉換要仔細算
- **相依**:無,任何時候都能做,不影響其他系統

## 二、中期(新系統,獨立模組,量體跟這次農耕差不多)

### 2.1 釣魚 🎣(建議優先)
- **為什麼**:玩法設計文件從第一版就列在「未來擴充」清單,是採礦、農耕之外第三種資源循環,能讓地圖生成多一種地形(水池/河流),視覺跟玩法都有新鮮感
- **做法草案**:
  - `genWorld()` 加一種可通行但特殊的地形 `T.WATER`(不可站,但可以站在岸邊對水面丟竿)
  - 新工具 `fishing_rod`(跟鏟子一樣是 `till`/`seed` 之外的第三種「非戰鬥工具」類別)
  - 簡化版判定:站在水邊右鍵,進條計時 + 隨機出魚(不用做完整小遊戲 QTE,參考現有「按右鍵→判定→結果」的既有模式,如 doTill/doPlant)
  - 魚類食材:接進料理系統(2.1 做完後直接有出路)
- **工程量**:中,新增地形生成邏輯(要確保水池不會擋住既有的隧道/礦脈可達性,這塊最容易出 bug,需要仔細測試)+ 新工具類別 + 掉落表
- **相依**:建議在 1.1 料理擴充之後做,魚才有地方用

### 2.2 天賦樹
- **現況**:升級只是單純 `LEVEL_CFG.hpPer`/`dmgPer` 固定加成,沒有選擇
- **做法草案**:每級(或每 2 級)彈出選擇視窗,3 選 1 被動加成(如「挖礦速度 +15%」「衝刺 CD -20%」「暗潮怪對星核傷害 -30%」),存進玩家物件的 `p.talents` 陣列
- **工程量**:中,UI 面板(仿現有背包/合成面板做法)+ 效果要接進對應的判定點(挖礦傷害算式在 `bestPick`、暗潮傷害在 `updateEnemies` 的星核攻擊那段等等,分散在好幾個檔案)
- **相依**:無,但要小心跟強化卷軸系統(ENH_CFG)的數值疊加不要爆掉,需要重新過一次戰鬥平衡

### 2.3 NPC 商人
- **做法草案**:星核旁安全區常駐一個不會動的 NPC,開一個交易面板(用戰利品/多餘礦物換稀有材料,如強化卷軸、鑽石),用固定匯率表
- **工程量**:中,UI 面板+簡單靜態實體(不需要 AI 移動)+ 交易表資料
- **相依**:無,建議在強化卷軸/鑽石之後做,才有「多餘資源換稀缺資源」的動機

### 2.4 動物養殖(牛羊雞豬)
- **做法草案**:圈養範圍(用圍籬 1.2 圍起來的區域判定)、動物實體(遊蕩 AI,比敵人簡單,不攻擊不追逐)、餵食(消耗作物/蘑菇)、產出(定時掉毛/蛋/奶,或宰殺掉肉)
- **工程量**:中大,需要新的「被動生物」AI 類別(現有 `updateEnemies` 是全部圍繞「追玩家攻擊」設計,被動生物要另外寫一套遊蕩+回應餵食的邏輯,不能直接沿用)
- **相依**:**建議排在農耕(圍籬)之後**,圈養需要圍籬先存在;產出食材建議接進料理系統(1.1)
- **待確認的設計問題**:
  - 地底遊戲怎麼解釋動物來源?(蛋/幼體用道具「孵化」,還是野外偶爾遇到可馴服的個體?)
  - 黑暗會生怪的機制,圈養區要不要也點光讓怪物不會生成在牲畜旁邊(現有光照防怪機制天然適用,可能不用額外處理)

### 2.5 工具/武器耐久度
- **現況**:所有鎬/劍/矛/鎚/弓弩/法杖都是永久耐用,不會壞
- **做法草案**:給 `ITEMS` 裡的 `pick`/`sword`/`ranged` 物品加一個 `durability` 欄位(最大耐久),背包格子上的裝備物件(`{id, count, lv, ...}`)多存一個 `dur` 目前耐久;每次 `doMine`(挖礦)、`doSwing`(揮擊)、`doShoot`(射擊)成功命中就扣 1 點,扣到 0 就損毀(從背包移除,可以做成直接消失或掉一個「壞掉的 XX」殘骸道具);UI 在快捷欄/背包格子的物品圖示下方加一條像血條的耐久條
- **工程量**:中,牽動的呼叫點不少(挖礦/近戰/遠程都要扣耐久並檢查損毀),但每一處改動都不大,風險可控
- **相依**:無,但要跟以下系統一起規劃數值,不然體驗會很痛:
  - **強化卷軸(ENH_CFG)**:裝備砸了好幾張卷軸衝到 +5,結果耐久用完直接消失,會讓玩家很挫折——建議衝裝時一併增加耐久上限,或乾脆衝到某個等級後耐久免疫
  - **修理機制**:要不要能在工作台花材料補耐久?(不做的話,耐久系統本質上是「裝備都是消耗品」,對玩家心理負擔較大,需要謹慎評估要不要做這麼硬核)
- **⚠️ 這是一個會改變遊戲手感的系統,建議先想清楚一個問題再開工**:目前微光深淵的裝備成長線是「打怪→掉卷軸→衝裝越打越強」的正向循環,耐久度是「用越多壞越快」的負向摩擦,兩者疊加要抓好平衡,否則後期玩家會花大量時間打材料修裝備而不是探索/防守,偏離核心玩法

## 三、長期(較大規模,會動到核心迴圈,建議前面幾項穩定後再排)

### 3.1 通關後內容
- 現在打完最終暗潮就結束(`gameOver(true)`),之後只能自由遊玩
- 選項 A:無盡模式(暗潮持續增強,類 roguelite 刷分/刷紀錄)
- 選項 B:解鎖第五區域(比黑曜區更深,更強的礦物/怪物)
- **工程量**:大,不管哪個選項都牽動 `WAVE_CFG`/`ENEMY_TYPES` 的無上限強度曲線設計,需要仔細調校

### 3.2 更多 Boss 種類
- 現況:三座神殿長得一樣,都是石像守衛
- 做法:三座神殿各自守不同 Boss(如冰系/火系/穿牆系,呼應現有元素相剋系統)
- **工程量**:中大(每隻 Boss 都要獨立 AI 行為 + 平衡),但**可以拆成三次獨立的小任務逐一上線**,不用一次做完

### 3.3 難度選項
- 開局選輕鬆/一般/困難,調整星核流失速度、暗潮強度倍率
- **工程量**:小中,主要是把 `CORE_CFG`/`WAVE_CFG` 的固定數值改成依難度縮放,UI 加一個開局選項
- **相依**:建議放在 3.1/3.2 都上線、整體數值曲線比較穩定後再做,不然每次調怪物數值都要重新過一次三種難度

### 3.4 多存檔/世界選擇
- 現況:`localStorage` 只認一份存檔(`gloamdepths_save` 這個固定 key),開新世界就會覆蓋掉舊的
- 做法:存檔 key 改成陣列/清單(如 `gloamdepths_save_1`、`_2`),主選單加「選擇存檔」畫面
- **工程量**:中,主要是選單 UI + 既有存讀檔函式的 key 從寫死改成參數化,风险較低但要仔细測試不要弄丟現有玩家的舊存檔(需要寫一個「偵測到舊格式存檔,自動搬進新格式」的相容邏輯)

### 3.5 鐵軌與礦車(快速移動)
- **做法**:新增一種可放置的軌道地形(跟木牆/石牆一樣的「放置地形」機制),站上去移動速度大幅提升(如 ×2.5~3),用來打通星核基地跟遠方礦區/神殿之間的快速通道
- **v1 建議做法(較簡單)**:軌道只是「加速地板」,不做真的會轉彎、有物理的礦車實體——移動邏輯照抄現有 `localControl` 的移速判定,偵測腳下 tile 是軌道就加乘速度即可
- **v2(進階,先不做)**:真正的礦車實體會沿軌道自動前進、遇轉轍器分岔——需要一套軌道連通圖 + 尋路邏輯,複雜度接近 Terraria 礦車/Factorio 火車軌道系統,不建議一開始就做
- **工程量**:中(v1)
- **相依**:無

### 3.6 傳輸帶 + 自動採礦
- **做法**:自動採礦機(放置建築,行為模式參考現有光塔/箭塔的「自動觸發」設計)靠近礦脈會定時自動採集(仍受鎬階級限制,金鎬機才挖得動鑽石礦);傳輸帶(跟軌道一樣是放置地形,但功能是把地上的掉落物往指定方向推)把礦物送回基地儲物箱
- **⚠️ 設計張力,開工前必須先決定**:全自動化會削弱「黑暗會生怪、玩家得主動冒險挖礦」這個核心迴圈的存在意義。建議的折衷方向(擇一或並用):
  1. 自動採礦機**只能架在玩家已經探索過、光照覆蓋的區域**(玩家還是得先冒險把地圖點亮才能佈署)
  2. 產量**遠低於手動**採礦(定位成「後期省事的 QoL」,不是「取代探索的效率解」)
  3. 自動採礦機本身也需要光晶持續供電才能運作,間接跟星核資源迴圈掛勾
- **工程量**:大(新地形類型 + 新建築 AI + 物品在傳輸帶上移動的渲染與邏輯,是這批裡最重的一項)
- **相依**:建議在 3.5 軌道做完、拿到「放置地形+高速移動」的經驗後再做,傳輸帶的物品搬運邏輯某種程度上是軌道機制的延伸

## 四、基礎建設(不是新玩法,是讓現有連線更穩)

### 4.1 房主斷線轉移
- 現況:房主關頁面,遊戲直接結束,其他人斷線
- 做法:偵測房主斷線後,其中一位客戶端升級成新房主(需要把當下的世界狀態從某個客戶端重建,或定期讓客戶端也持有一份唯讀快照)
- **工程量**:大,牽動整個連線架構的「房主權威」假設,是目前系統風險最高的一項改動

### 4.2 TURN 中繼伺服器
- 解決極少數嚴格 NAT 網路環境連不上的問題(玩法設計文件本來就列為已知限制)
- **工程量**:中,主要是設定成本(要找免費/低成本 TURN 服務或自架),程式面只要 PeerJS 設定加一組 ICE server 設定值,改動很小

### 4.3 觀戰模式
- 死亡或想旁觀時可以自由移動鏡頭看隊友
- **工程量**:小中,`render.js` 的攝影機邏輯要能脫離 `myPlayer()` 位置,新增一個觀戰狀態機

## 建議排序(僅供參考,四個大方向都值得做)

```
第一批(短期,先把地基墊好):
  1.1 料理擴充 → 2.1 釣魚(魚馬上有地方用)→ 1.2 圍籬(順手做,農耕/未來畜牧都用得到)

第二批(選一個中期系統深入做):
  2.2 天賦樹  或  2.4 動物養殖(圍籬做完後就有基礎了)

第三批(視第一二批的手感決定要不要繼續加深度):
  2.5 耐久度(先想清楚跟衝裝系統的平衡)→ 2.3 NPC 商人 → 3.2 更多 Boss(可拆三次上) → 3.3 難度選項

第四批(長期投資,建議最後排):
  3.5 鐵軌快速移動 → 3.6 傳輸帶+自動採礦(3.6 開工前務必先決定設計張力的折衷方案)
  → 3.1 通關後內容 → 3.4 多存檔 → 4.1 房主轉移
```

1.4(大地圖)、4.2(TURN 中繼)工程量都很小、不影響其他系統,不受這個排序限制,想插隊隨時可以做。
